# Plan Detail UX 리뷰 결과

검토 기준 파일:
- `PLAN_DETAIL_UX_REVIEW.md`
- `src/pages/PlanDetailPage.tsx`
- `src/components/DayView.tsx`
- `src/components/ScheduleCard.tsx`

## A. 상단 레이아웃 정리
판정: MODIFY

이유:
- 현재 `메인 탭(일정/메모/앨범)` + `일정 제목 + 뷰 탭 + 일정 추가 버튼` 구조가 정보 밀도가 높은 것은 사실입니다.
- 다만 `일정` 텍스트를 제거하고 `+ 추가`를 아이콘 전용으로 바꾸는 안은 가시성과 접근성을 동시에 떨어뜨릴 수 있습니다.
- 권장안은 `일정` 헤더 제거는 수용하되, 추가 버튼은 아이콘+텍스트(축약형) 또는 모바일/데스크톱 반응형 라벨 유지입니다.

## B. 상단 지도 + DayView 연동
판정: MODIFY

이유:
- 지도 중복 문제는 실제로 존재합니다. 현재 DayView 내부 지도와 상단 지도가 동선 정보를 이중으로 보여줍니다.
- `daily` 모드에서 상단 지도를 날짜 필터링하는 방향은 타당합니다.
- 다만 DayView에서 지도를 완전 제거하면, 상단 지도 collapse 상태/스크롤 위치에 따라 일별 맥락이 약해질 수 있습니다.
- 권장안은 "daily 모드에서는 상단 지도만 일별 필터"를 기본으로 하고, DayView에는 지도를 제거하되 "오늘 동선 n곳" 같은 요약 텍스트는 남겨 맥락을 보완하는 방식입니다.

## C. 목록(vertical) 뷰 날짜 구분자
판정: APPROVE

이유:
- 현재 vertical은 카드 자체 날짜만 있고 리스트 단위 구분이 없어 스캔성이 낮습니다.
- 날짜 헤더(`Day n · 날짜`)를 넣으면 탐색 속도와 인지 부하가 확실히 개선됩니다.
- 기존 `groupedSchedules`가 이미 있어 구현 복잡도도 낮습니다.

## D. 타임라인 compact 카드
판정: MODIFY

이유:
- 현재 horizontal에서 `ScheduleCard`가 동일 크기로 렌더링되어 한 화면 정보량이 적은 문제는 맞습니다.
- compact 모드 도입은 효과적입니다.
- 다만 폰트/배지/패딩을 과도하게 줄이면 터치 타깃과 가독성이 떨어집니다. 특히 모바일에서 카드 클릭 영역 최소 높이 보장이 필요합니다.
- 권장안은 `compact`를 단순 축소가 아닌 "정보 우선순위 재배치(시간/제목/장소 우선, 메모 1~2줄)"로 설계하는 것입니다.

## E. DayView 인디케이터 개선
판정: APPROVE

이유:
- 현재 도트(`w-2.5 h-2.5`)는 대비와 식별성이 낮아 실제 사용성이 떨어집니다.
- pill 형태 `D1 D2...`는 선택 상태 인지와 터치 정확도를 동시에 개선합니다.
- 일정 없음/메모 있음 상태를 opacity나 배지로 구분하는 방향도 일관적입니다.

## F. 일정 추가 폼 (AI 입력 textarea + 디자인 정리)
판정: MODIFY

이유:
- 현재 AI 입력이 단일 `<input>`이며 자연어 다줄 입력 UX와 맞지 않습니다.
- textarea 전환은 타당합니다.
- 다만 기존 `Enter` 즉시 생성 플로우를 그대로 두면 줄바꿈과 충돌합니다. `Ctrl/Cmd+Enter 생성` 또는 버튼 중심 제출로 상호작용 규칙을 재정의해야 합니다.
- 폼은 이미 일정 수준 정리되어 있으므로 "전체 재디자인"보다 입력 행태와 피드백 상태(로딩/성공/실패) 정돈에 집중하는 게 효율적입니다.

## 주요 리스크
- `daily` 날짜 상태가 `DayView` 내부 state라 상단 지도 필터 연동 시 상태 소유권 충돌 가능성.
- 지도 단일화 시 collapse 상태에서 사용자에게 "지도가 사라졌다"고 느껴질 가능성.
- compact 카드에서 텍스트/타깃 축소 과다 시 모바일 터치 오류 및 접근성 저하.
- AI 입력 textarea 전환 시 키보드 단축키 규칙 미정으로 오동작(의도치 않은 제출/줄바꿈 실패) 위험.

## 우선순위 (높음 → 낮음)
1. E (Day 인디케이터 가시성 개선)
2. C (vertical 날짜 구분자)
3. D (horizontal compact 카드)
4. B (지도 단일화 + daily 연동)
5. F (AI textarea + 입력 규칙 정리)
6. A (상단 레이아웃 미세 정리)

## 추가 UX 개선 제안
- `daily` 진입 시 선택 날짜를 URL 쿼리 또는 상위 state로 동기화해 새로고침/공유 시 동일 날짜 복원.
- 뷰 전환 탭(`목록/타임라인/일별/캘린더`)에 아이콘+텍스트를 통일하고 `📅` 단독 라벨은 명시 텍스트로 변경.
- `vertical/horizontal` 공통으로 "오늘 일정으로 점프" 액션 제공.
- `ScheduleCard`의 상태색(`upcoming/current/past`) 대비를 WCAG 기준에 맞게 점검.
- 일정 추가 모달에서 AI 생성 결과를 바로 저장하지 않고, 1회 검토 단계(미리보기) 제공.
